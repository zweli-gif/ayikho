# Ayikho App UX/UI Redesign Prompt

**Role:** Expert UX/UI Designer and Front-End Developer

**Objective:**
I have an educational application called "Ayikho," built for South African TVET college students. Currently, it consists of several HTML prototype files (e.g., `app.html`, `ayikho-quiz.html`, chapter files). My goal is to uplift the overall user experience and interface (UX/UI) to make it feel premium, engaging, and highly intuitive for students learning on mobile devices.

Please review the provided HTML file(s) and provide a comprehensive UX/UI audit along with a functional HTML prototype demonstrating your proposed improvements.

## Instructions:

### 1. UX/UI Audit
Please analyse the provided code focusing on:
*   **Information Architecture:** Is the navigation logical? (Home dashboard vs. Quiz vs. Modules)
*   **Visual Hierarchy:** Do the most important elements (like the "Today's Lesson" call to action or streak counters) stand out natively?
*   **Aesthetics & Branding:** The app uses distinct color themes for different courses (e.g., Green for Accounting, Blue for Communication, Purple for IT). Evaluate how these are applied. Does the app look modern, sleek, and premium?
*   **Micro-interactions:** Are there opportunities for engaging animations (hover states, pulsing buttons, completion celebrations)?
*   **Accessibility & Mobile-First Design:** Is the layout fully responsive and legible on a standard mobile device screen?

### 2. Deliverable Structure
Provide your response in the following format:
1.  **Strengths:** What currently works well in the design?
2.  **Areas for Improvement (The Audit):** Detailed bullet points explaining what needs fixing and *why* from a UX perspective.
3.  **Proposed Design System:** Briefly outline your suggested typography, color palette refinements, and component styling (e.g., card border radii, shadow depths).
4.  **Refactored HTML Prototype:** Provide a single, self-contained HTML file (using embedded Vanilla CSS and JS) that implements your redesign.

### 3. Requirements for the HTML Prototype:
*   **Self-Contained:** It must be a single `.html` file I can open in the browser to see the result immediately. Do not provide a PDF; a functional HTML prototype is much easier for me to test and integrate.
*   **Aesthetics:** Use modern CSS techniques (glassmorphism, soft overlapping shadows, smooth gradients, CSS grid/flexbox) to make it look stunning. Do not use generic colors; use curated HSL/Hex palettes.
*   **Interactivity:** Include basic JavaScript to demonstrate interactive elements, such as switching between course themes or toggling a menu.
*   **No External Frameworks:** Use plain HTML, CSS, and JS (Tailwind or React are not needed for this prototype unless absolutely necessary for your specific vision, but Vanilla is preferred).
